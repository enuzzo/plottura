Design Section 2: Map Style & Rendering

How the 3D features integrate into generateMapStyle() and MapPreview

A. Building layer: fill → fill-extrusion

Currently the building layer is a flat fill. When enable3D && buildingExtrusion, we replace it with a fill-extrusion layer using the same color from the theme.

// In generateMapStyle() — new option: enable3D, buildingExtrusion

// When 3D is OFF (current behavior):
{ id: "building", type: "fill",
  paint: { "fill-color": buildingFill, "fill-opacity": 0.84 } }

// When 3D is ON:
{ id: "building-3d", type: "fill-extrusion",
  source-layer: "building",
  paint: {
    "fill-extrusion-color": buildingFill,
    "fill-extrusion-height": ["get", "render_height"],
    "fill-extrusion-base": ["get", "render_min_height"],
    "fill-extrusion-opacity": 0.84,
    "fill-extrusion-vertical-gradient": true
  } }

💡 render_height and render_min_height come from OpenMapTiles vector tiles — already available in your OpenFreeMap source. vertical-gradient darkens the base, adding subtle depth.

B. Light object in style spec

MapLibre's root-level light property controls how fill-extrusion layers are shaded.

// Added to the StyleSpecification return when enable3D is on

light: {
  anchor: "map",            // light rotates with map bearing
  color: "#ffffff",
  intensity: 0.6,          // from state.form.lightIntensity
  position: [1.15, 210, 30]  // [radial, azimuth, polar]
                               // azimuth from state.form.lightAzimuth
}

C. Pitch control in MapPreview

MapPreview.tsx currently doesn't set pitch. We add it as a prop:

MapPreview changes

New pitch prop (number). Passed to the MapLibre Map constructor and synced via map.setPitch() when the value changes. When enable3D is off, pitch = 0.

PreviewPanel wiring

Reads form.enable3D and form.mapPitch. Computes pitch = enable3D ? Number(form.mapPitch) : 0. Passes to MapPreview.

D. Terrain (Phase 3b — later milestone)

Terrain requires adding a raster-dem source and calling map.setTerrain(). Shipped as a separate milestone because:

DEM tile source needed

Best free option: MapLibre demo tiles (low-res) or MapTiler free tier. We need to evaluate quality for print at 300 DPI.

Independent from buildings

Terrain works independently — can ship buildings + pitch first, add terrain later without touching the building code.

E. Integration with incremental style updates

Your MapPreview already does incremental style diffs via applyIncrementalStyleUpdate(). The 3D changes flow through the same mechanism:

Change How it flows
Toggle 3D on/off generateMapStyle() regenerated → full style swap (layer type changes from fill to fill-extrusion)
Change pitch Direct map.setPitch() — no style regeneration
Change light map.setLight() — no style regeneration needed
Change theme colors Incremental paint update on fill-extrusion-color — existing diff mechanism
