Design Section 1: State & Data Model

New fields added to PosterForm in posterReducer.ts

New PosterForm fields

// ── 3D View ──
enable3D: boolean   // master toggle (default: false)
mapPitch: string    // camera tilt angle "0"–"85" (default: "60")
buildingExtrusion: boolean   // 3D extruded buildings (default: true)
terrainEnabled: boolean   // DEM terrain relief (default: false — needs tile source)
terrainExaggeration: string   // terrain scale "0.5"–"3.0" (default: "1.5")
lightAzimuth: string   // light direction "0"–"360" (default: "210")
lightIntensity: string   // light strength "0.0"–"1.0" (default: "0.6")

Why these choices

String for numeric fields

Follows existing convention — width, height, latitude, distance are all strings in PosterForm, parsed to numbers where needed.

enable3D as master toggle

When off, pitch=0 and buildings stay flat (current behavior). When on, pitch applies + extrusion layers activate. Single switch flips the whole experience.

terrain defaults to off

DEM tiles need an external source. We can ship buildings first and add terrain later without breaking anything.

Separate building extrusion toggle

User might want tilted map without extruded buildings (e.g., terrain-only for mountain posters). Or buildings without terrain for city posters.

Behavior matrix

enable3D buildingExtrusion Result
❌ off — Current 2D flat view (pitch=0, fill layer for buildings)
✅ on ✅ on Tilted map + extruded buildings + light shading
✅ on ❌ off Tilted map only (good for terrain-only or minimal look)
