Three approaches for 3D isometric map

All keep the poster flat — only the map tilts. Ranked by my recommendation.

★

A. MapLibre Native (Recommended)

Use MapLibre's built-in pitch, fill-extrusion layers, terrain DEM, and light object. Zero new dependencies. The same engine that renders your 2D map renders the 3D — same tile source, same theme colors, same export pipeline.

Pros
  • Zero new dependencies
  • Same tile source (OpenFreeMap)
  • Theme colors apply automatically
  • Export already captures pitch
  • Buildings from existing vector tiles
  • Terrain via free DEM tiles
  • Lighting built into style spec
Cons
  • No real shadows (only directional shading)
  • Lighting is basic (angle + intensity only)
  • Less creative control vs full 3D engine
  • Terrain needs external DEM tile source
B

B. Three.js Custom Layer on MapLibre

MapLibre renders the base map (roads, water, land). A Three.js custom layer adds 3D buildings with proper lighting, shadows, and materials on top. Hybrid approach — the example you found.

Pros
  • Real shadows on buildings
  • Full material/lighting control
  • Custom post-processing possible
  • Still uses MapLibre for base map
Cons
  • +150KB (Three.js)
  • Must extract building geometry from tiles
  • Complex camera sync (matrix math)
  • Two renderers sharing one WebGL context
  • Theme color sync is manual
  • Terrain still needs MapLibre or separate DEM parsing
C

C. Deck.gl Overlay

Deck.gl is a geospatial visualization library by Uber. Has a TerrainLayer and GeoJsonLayer with extrusion. Designed to overlay on MapLibre. Purpose-built for this.

Pros
  • Purpose-built for geo 3D
  • Nice terrain layer
  • MapLibre integration plugin exists
  • Good lighting system
Cons
  • +200KB dependency
  • Separate rendering pipeline
  • Overkill for flat-colored extrusions
  • Export integration unclear
  • Less community examples for poster-style use

Why I recommend A (MapLibre Native)

Your goal is a minimal, flat-colored isometric view — not photorealistic 3D. MapLibre's fill-extrusion produces exactly that aesthetic: clean geometric volumes, themed with your existing palette. The pitch + directional light shading gives depth without complexity. Your export pipeline already captures pitch. The buildings come from the same OpenFreeMap tiles you already use. And terrain can be added later with a DEM source.

Three.js (B) and Deck.gl (C) add real value when you need custom materials, real shadows, or complex 3D geometry. For flat-colored buildings in perspective? MapLibre does it natively in ~50 lines of style config.
