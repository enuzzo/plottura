Vibemilk v3.0.0

Vibemilk - the Netmilk Studio Design System

Yes, another design system. No, not the usual mess of random classes and 4MB CSS files. Vibemilk is built for Netmilk web interfaces of every flavor: fast, readable, token-driven, and ready for humans and AI to use without pain by the third component.

Brand: Netmilk Studio Active theme: Vibemilk Default Class namespace: .vm-* Architecture: core + modular themes
Official Structure

File system designed to scale to dozens of themes

Core rule: the core layer does not know custom themes. Themes only override tokens and, when needed, visual micro-tweaks.

Quick start

Integration in a Netmilk web UI project

Mode A: Single Theme (lighter)

Load only core + one theme file. Zero overhead from unused themes.

<link rel="stylesheet" href="css/vibemilk-core.css">
<link rel="stylesheet" href="css/themes/cyber-grid.css">

<html data-theme="cyber-grid">
Mode B: Runtime Theme Switcher

Core is always loaded, themes are injected on demand by the JS manager.

<link rel="stylesheet" href="css/vibemilk-core.css">
<script src="js/vibemilk-docs.js" defer></script>

<select data-vm-theme-select>...</select>
Standard storage key: vibemilk.design-system.theme. If a requested theme is missing, fallback is automatically set to Vibemilk Default.

Playbook AI

Explicit instructions for agents integrating Vibemilk

Operating rules
  • Use only the .vm-* namespace for components.
  • Do not hardcode colors: always use --* tokens.
  • For single-theme mode, load only the required theme file.
  • For runtime switching, use the JS manifest and lazy-load theme files.
  • Avoid direct component overrides: prefer token overrides.
AI output checklist
[ ] integration mode selected (single/switcher)
[ ] coherent data-theme set
[ ] minimum required files imported
[ ] correct vm classes used
[ ] no dependency on legacy names
[ ] basic accessibility covered (label, focus-visible, contrast)
Recommended convention for new custom components: .vm-{feature} + BEM variants like __element, --variant.
Elementor V4

Using Vibemilk with Elementor V4

Elementor V4 introduced a CSS-first architecture with Variables Manager, Global Classes, and Scoped Custom CSS. This maps directly to how Vibemilk works.

How Vibemilk Maps to Elementor V4

VibemilkElementor V4Where
--accent-primary: #7551FF Variable: accent-primary Variables Manager (Color type)
--font-family: 'Montserrat'... Variable: font-family Variables Manager (Typography type)
--space-4: 16px Variable: space-4 Variables Manager (Size type)
.vm-btn Global Class: vm-btn Global Classes panel
.vm-btn--primary Global Class: vm-btn--primary Global Classes panel
.vm-card Global Class: vm-card Global Classes panel

Quick Start: 3 Steps

Step 1: Import Variables

Open Elementor's Variables Manager (Site Settings → Design System → Variables). Create a variable for each Vibemilk token you need.

Variable Name: accent-primary
Type: Color
Value: #7551FF

Variable Name: font-family
Type: Typography
Value: Montserrat, system-ui, sans-serif

Variable Name: space-4
Type: Size
Value: 16px

Use tokens/{theme}.elementor.json as reference — run npm run build:tokens to generate.

Step 2: Create Global Classes

For each Vibemilk component, create a Global Class in Elementor and paste the scoped CSS into its Custom CSS field.

/* Global Class: "vm-btn--primary" */
/* Paste into Custom CSS field: */
selector {
  background: var(--accent-primary);
  color: var(--text-on-primary);
  height: var(--btn-height-md);
  padding: 0 var(--btn-padding-md);
  border-radius: var(--radius-sm);
  font-weight: var(--fw-semibold);
  border: none;
  cursor: pointer;
  transition: var(--transition-fast);
}
selector:hover {
  filter: brightness(1.1);
}

Elementor V4 uses selector as the scoping placeholder for Custom CSS in Global Classes.

Step 3: Switch Themes

To switch themes in Elementor, update the Variable values in the Variables Manager. All Global Classes referencing those variables will update automatically across every page.

vibemilk-default:

accent-primary: #7551FF
bg-deepest: #0D0B14
text-primary: #F0ECF9

cyber-grid:

accent-primary: #00FFD1
bg-deepest: #0A0E17
text-primary: #E0F4FF

One set of Global Classes, infinite themes. Update Variables, the whole site follows.

Token Export for Elementor

The build script generates an Elementor-ready JSON file for every theme:

npm run build:tokens
# Outputs: tokens/{theme}.elementor.json for all 10 themes

Each .elementor.json contains a flat list of variables with name, type, and value — ready for manual entry into the Variables Manager:

{
  "meta": {
    "theme": "vibemilk-default",
    "version": "3.0.0",
    "format": "elementor-v4-variables"
  },
  "variables": [
    { "name": "Bg Deepest", "type": "Color", "value": "#0D0B14" },
    { "name": "Accent Primary", "type": "Color", "value": "#7551FF" },
    { "name": "Font Family", "type": "Typography", "value": "'Montserrat'..." },
    { "name": "Space 4", "type": "Size", "value": "16px" }
  ]
}

Copy for Elementor (Component CSS)

On the Components page, use View Source on any component demo to see its HTML. The CSS for each component is defined in the css/components/ directory. To create an Elementor Global Class, copy the relevant CSS rules and wrap them with selector { ... } scoping. Key components:

Pro tip: Start with the 5-10 core tokens (accent colors, bg, text, font-family, spacing) and the 3-4 most-used component classes. Expand as needed. You don't need to import all 270+ tokens on day one.
Icons

Nerd Font Glyphs

Vibemilk ships 10,000+ icon glyphs via Nerd Fonts (dev icons, powerline, Material Design, Font Awesome, Octicons, Weather, and more). These are icons, not text typefaces -- text fonts (Montserrat, Space Mono, etc.) are a separate concern.

1. Loading the icon font

Production (subset)

Just load vibemilk-core.css. It already imports icons.css, which declares the @font-face for VibemilkIcons.woff2 (~50-80 KB). This subset covers the most common glyphs used in Vibemilk components.

<link rel="stylesheet" href="css/vibemilk-core.css">
<!-- icons.css + @font-face included automatically -->
Full Nerd Font

Both VibemilkIcons.woff2 (subset) and SymbolsNerdFontMono-Regular.woff2 (full, ~1.2 MB) are registered via @font-face. The browser loads the full font only when a glyph outside the subset is requested.

/* Both are declared in css/components/icons.css */
@font-face {
  font-family: 'VibemilkIcons';
  src: url('../fonts/VibemilkIcons.woff2');
}
@font-face {
  font-family: 'NerdFontSymbols';
  src: url('../fonts/SymbolsNerdFontMono-Regular.woff2');
}

2. The --font-icons token

Defined in :root, theme-agnostic. All themes inherit the same icon font stack. The fallback chain ensures graceful degradation:

:root {
  --font-icons: 'VibemilkIcons', 'NerdFontSymbols', 'Symbols Nerd Font Mono', sans-serif;
}

3. Using icons

Unicode-first

Paste the Unicode codepoint directly. Best for one-off icons or dynamic content.

<span class="vm-nf" aria-hidden="true">&#xf1d3;</span>
Named class

Use a vm-nf-{name} class. The glyph is injected via CSS ::before. Clean markup, no Unicode in HTML.

<span class="vm-nf vm-nf-git" aria-hidden="true"></span>
With modifiers

Combine base class + size modifier + color modifier. All modifiers are optional and composable.

<span class="vm-nf vm-nf--lg vm-nf--accent" aria-hidden="true">&#xf1d3;</span>

5. Color modifiers

  • .vm-nf--accent -- uses var(--accent-primary)
  • .vm-nf--secondary -- uses var(--accent-secondary)
  • .vm-nf--muted -- uses var(--text-muted)
  • .vm-nf--success -- uses var(--color-success)
  • .vm-nf--warning -- uses var(--color-warning)
  • .vm-nf--danger -- uses var(--color-danger)

6. Accessibility

Decorative icons

Icons that are purely visual (next to a text label, or for decoration) must be hidden from assistive tech:

<span class="vm-nf vm-nf-git" aria-hidden="true"></span> Git
Informational icons

Icons that convey meaning on their own need an accessible label:

<span class="vm-nf vm-nf-warning" aria-label="Warning"></span>

<!-- or with sr-only text -->
<span class="vm-nf vm-nf-warning" aria-hidden="true"></span>
<span class="sr-only">Warning</span>

7. ESP32 / Arduino

The web font is not applicable on microcontrollers. For ESP32 TFT displays using TFT_eSPI or similar libraries, use the codepoint values from the C header token export (tokens/{theme}.h). Each icon's Unicode value can be rendered as a text character if the display font supports it.
